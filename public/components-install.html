Installation

If you would like to install crux globally and use it as a tool to generate project boilerplates,

npm install -g node-crux

This will add the crux executable to the system's path, allowing you to use crux's CLI to install various components. You could do so by calling

crux install [list of components to install]
Available components:

  • scss - Installs the SCSS build system and a basic web server for serving the generated static content
  • less - Installs the LESS build system and a basic web server for serving the generated content
  • angular - Installs the Angular build system, along with a basic web server for serving the built content
  • cjs - Installs the CommonJS + Templates build system, along with a basic web server
  • redis - Installs the standalone redis component. This is to be used along with at least one other component
  • mongo - Installs the standalone mongodb wrapper. This is to be used along with at least one other component
  • sql - Installs the standalone sql wrapper. This is to be used along with at least one other component.
  • server - Installs the crux http server
  • service - Installs the service-layer component. This is to be used with at least one other component.
  • static - Installs the express static server

A few application bootstrap examples...

crux install angular less - creates an application structure suitable for front-end applications, equipped with less watcher and compiler along with angular's boilerplate application structure, allowing the developer to focus on the application, and not on how to make things work and tie them up together.

crux install server sql service - creates a full back-end project suitable for API applications that use MySQL as their database.

crux install server mongo redis - creates a full back-end project suitable for applications that use the Redis store and MongoDB in their stack

For a more complete view on Crux-powered applications, you can visit the Example projects section.
