Crux comes in a simple, complete package that contains several components. You may choose to install them all or only the ones you believe you’ll need. Don’t worry, you’ll be able to add components at any time.

What is it and what it does?

crux install angular
The installer will generate an angular boilerplate structure along with the project's configuration and components.
Angular JavaScript watcher, concatenates the JavaScript files into a single build file, with custom include patterns and environment-specific configurations.
Angular Template watcher, caches the HTML contents of each template into a single JavaScript file, used by Angular at runtime to fetch the views.
By default, installing the angular component will also install the crux static server for asset serving
This is commonly used together with less or sass components
See API Documentation.

crux install cjs
Commonjs watcher, uses browserify & aliasify to provide commonjs require() functionality, equipped with dynamic require paths and an integrated watcher for auto-building.
The component also includes a build process for any kind of front-end templates (html, swig, etc), that will be cached under a window variable (ex: window['views'][filePath] = htmlContent). This allows for easy client-side cached templates.
By default, installing cjs will also install the crux static server for asset serving
This is useful for javascript apps that do not use any external framerowk and handle their own business. It is also commonly used together with less or sass
See API Documentation.

crux install less
Less watcher, used to easily compile less files to a CSS output. This component also comes with an integrated static server and a file-watcher for auto-compiling.
See API Documentation.

crux install sass
Sass watcher, same as less, but for scss files.
See API Documentation.
