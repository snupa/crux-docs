crux.Build. Angular

The crux angular build process provides angular-related building process. As most angular applications tend to be stored into a single javaScript file placed on a CDN, this component facilitates the process, watching and building (concatenating) all angular files found under the root project directory
The process is divided into 3 separate sub-processes that can all or some be included:

  • crux.Build.Angular.Core - handles javaScript concatenation
  • crux.Build.Angular.Templates - handles html caching with angular's $templateCache service
  • crux.Build.Angular.Index - watcher over the index.html root file and copies it into the public folder.

    Note: the main angular process still inherits from crux.Build.Interface but it is more of a sub-process manager over all of angular's subprocesses.
    Note 2: This class should not be overridden or modified in any way, as it may cause unstability.
