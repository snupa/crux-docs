<abstract, static> component#attach(methodName, methodFn)

Because we want each component to be aware of other components, we need an injection system, capable of attaching functions from one component inside another component.
Components that allow attaching (or injecting) functionality into other components will have to implement the attach() function.

<abstract, static> component#init()

This is the second function that is called in the creation cycle of a component.
The cycle is:
-> new Component() (creates all components)
-> components.init() (initialize all, after creation)
-> components.run() ( runs all components, after init)

<abstract, static> component#packages() → {Array.<string>}

Each component can define its individual npm package requirements to be installed, before the actual initialization of the component. This is required to be overridden by all components that require external dependencies.
The packages are installed BEFORE we call their init() method.
Each individual package name must have the following format:
[name]@[version] - if version is not specified, we will use latest</b.
