Class: Sql

crux.Database. Sql

new Sql(options)

The SQL adapter component wraps itself over Sequelize to offer standardized model-definition as well as module auto-loading and automatic relationship building.
For now, this component is optimized only for MySQL but in the future it will support all of Sequelize's engines
Note: the SQL scripts path may contain $environment in its path. Should this happen, the component will dynamically parse the path at runtime, replacing $environment with the current environment value.
As an example, if environment=dev and path.sql=app/models/sql/$environment, when in setup mode, it wiill read all .sql files from app/models/sql/dev and execute their statements.
Note: the SQL script content must contain one SQL statement per line

<static> init()

This will initialize the database component, loading up all model definitions under the models/ path
It is now when the model relations are also created and parsed. An important convenient feature is if it finds a --setup=sql in the process arguments, we will automatically enable the setup process, thus re-creating the entire database.

<static> run(done)

Initializes and configures the sequelize instance. At this point, it will try and connect to the MySQL server with the configured credentials. Once the connection is established, it will try to synchronize the models and if it was configured to perform database setup, it will drop all existing tables and re-create them based on the model definition files (models + their relationships and lookup tables).
After the setup process has been completed, it will look into the custom sql scripts path and try execute them.
