What is it?

The most hardcore component, which uses express at its core.

See API Documentation.

What it does?

Promisified request chaining, with namespaced middleware (known as checkpoints) available in any route definition

Clear pre-requisites variables for every endpoint (querystring data, body data, header data, URL parameters) with built-in validations for various variable types

Functionality injection from other components, in each request.

Extensible route definition chaining (ex: Route.get('/').then().then().error());

Auto-generation of HTTP paths, relative to the JavaScript file path.

Utility functions for the view-layer (currently twig)

Extensible classes: Request, Route, Chain, all 3 offer base functionality

Auto-loading routes

Auto-generation for route documentations.

Highly customizable options

Definition of security checkpoints that use .allow() and .fail()

Variable persistence from one checkpoint to another by passing their reference in the next arguments scope
